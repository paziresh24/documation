<FAQBox title="نکات امنیتی API Key چیست؟">
مسوولیت هرگونه سوءاستفاده از کلیدهای API شما، بر عهده‌ی خود شماست. برای حفظ امنیت کلیدها به نکات زیر توجه کنید:

- کلید را در هدر `Authorization` قرار دهید. درخواست‌های بدون کلید رد خواهند شد.

- _کلیدها را در کد ذخیره نکنید:_ از متغیرهای محیطی (Environment Variable)، فایل‌های کانفیگ یا ابزارهای مدیریت کلید استفاده کنید.

- _کلیدها را در git ذخیره نکنید:_ اگر کلیدها را در فایل‌های کانفیگی نگه می‌دارید، مطمئن شوید این فایل‌ها توسط git (یا سایر ابزارهای مدیریت ورژن) کنترل نمی‌شوند. برای مثال، اگر از git استفاده می‌کنید، حتما این فایل‌ها را به `.gitignore` پروژه‌ی خود اضافه کنید.

</FAQBox>
